## Issues acknowledged, not fixed (with rationale)

- **`hope_block.py` forward substitution loop is `O(C)` in Python.**  
  At `chunk_size=16` and CIFAR `S=65`, this is fine. Vectorising via
  `torch.linalg.solve_triangular` is a worthwhile follow-up if/when the
  model is ported beyond CIFAR-scale, but it touches a tested numerical
  path so we deferred it to a separate change.

- **`kaggle_cf_test.py` duplicates the model code instead of importing.**
  These are notebook-style scripts intended for one-shot Kaggle execution.
  We left them alone but added the cleaner, importable
  `cf_benchmark.py` as the canonical entry point.

- **`auto_device()` silently falls back to CPU when MPS is missing.**
  This is the documented behaviour; the smoke test confirms it works on
  MPS. We did not change it.
